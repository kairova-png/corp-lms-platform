# Corp LMS Platform

Переиспользуемый **дистрибутив корпоративной LMS на базе Moodle 4.5 LTS** — преднастроенный,
забрендированный, с бандлом плагинов и собственными расширениями под требования
корпоративных тендеров РК (Самрук-Казына / KMG PetroChem и аналогичные).

## Зачем
ТЗ таких тендеров (SCORM/xAPI, продвинутый движок тестов, AD/LDAP/SSO/2FA, сертификаты,
вебинары, отчёты, on-prem, ИБ/OWASP) на ~85–90% закрывает зрелый Moodle. Мы **не пишем LMS
с нуля**, а собираем продуктовую сборку: ядро Moodle + проверенные плагины + наши кастом-плагины
под недостающее (бюджетирование обучения, IDP/360°, Киркпатрик L3–L4, коннекторы). Одна
сборка переиспользуется из тендера в тендер = recurring-внедрения.

## Состав
| Слой | Что |
|---|---|
| `docker/` | Воспроизводимый стек: Moodle (PHP 8.3) + PostgreSQL + Redis. On-prem-ready |
| `config/plugins.json` | Манифест бандла сторонних плагинов (тип, путь, git, ветка) |
| `scripts/install_plugins.sh` | Стейджит плагины из манифеста в дерево Moodle перед сборкой |
| `moodle/plugins/local/corplms` | **Наш** local-плагин — дом для кастом-функций под гэпы ТЗ |
| `moodle/theme/` | Брендированная тема (child of Boost), KZ/RU/EN |
| `docs/` | Архитектура, мэппинг на ТЗ, runbook развёртывания |

## Быстрый старт (на машине с Docker)
```bash
# 1. подготовить переменные
cp .env.example .env        # отредактировать пароли/домен

# 2. стейджинг плагинов из манифеста
bash scripts/install_plugins.sh

# 3. поднять стек
docker compose -f docker/docker-compose.yml up -d --build

# 4. установка Moodle (CLI), затем открыть http://localhost:8080
docker compose -f docker/docker-compose.yml exec app \
  php admin/cli/install_database.php --agree-license --adminpass="$MOODLE_ADMIN_PASS" --adminemail="$MOODLE_ADMIN_EMAIL"
```
