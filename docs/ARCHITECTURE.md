# Архитектура развёртывания (on-prem)

ТЗ требует on-premises в контуре заказчика, без облака (п.12), 24/7/365, до 500 одновременных
пользователей, доступность ≥99.5%/мес, восстановление ≤4ч.

## Прод-топология (кластер)
```
                       ┌──────────────┐
        HTTPS  ───────►│ Load Balancer │  (Nginx/HAProxy, TLS 1.2+ терминация)
                       └──────┬───────┘
                ┌─────────────┴─────────────┐
          ┌─────▼─────┐               ┌─────▼─────┐
          │  app-1    │               │  app-2    │   Moodle (PHP-FPM), stateless
          │ (Moodle)  │               │ (Moodle)  │   сессии/кэш → Redis
          └─────┬─────┘               └─────┬─────┘
                └───────────┬───────────────┘
              ┌─────────────┼───────────────┐
        ┌─────▼─────┐ ┌─────▼─────┐   ┌──────▼──────┐
        │ PostgreSQL│ │   Redis    │   │ Файловое    │
        │ primary + │ │ кэш+сессии │   │ хранилище    │  (moodledata, NFS/объектное)
        │  replica  │ └────────────┘   └─────────────┘
        └───────────┘
        ┌───────────────┐   ┌──────────────────┐
        │ BigBlueButton │   │ Мониторинг        │  Prometheus/Grafana или Zabbix
        │  (вебинары)   │   │ + бэкап по cron   │
        └───────────────┘   └──────────────────┘
```

- **Stateless app-ноды** за балансировщиком → горизонтальное масштабирование и горячее
  резервирование (п.12.1). Сессии и кэш вынесены в Redis.
- **PostgreSQL primary+replica** → отказоустойчивость и чтение-масштабирование.
- **moodledata** на сетевом/объектном хранилище, общем для app-нод.
- **BigBlueButton** (ядро Moodle) или Zoom/Teams для вебинаров (≥100 слушателей, п.5.4).
- **Бэкапы по расписанию** (БД + moodledata) + **мониторинг доступности** (SLA 99.5%).

> Dev/демо-вариант (`docker/docker-compose.yml`) — упрощённый одно-нодовый стек для разработки
> и демонстраций. Прод собирается из тех же образов в кластер средствами оркестратора заказчика.

## Соответствие ИБ (п.12.2)
| Требование ТЗ | Реализация |
|---|---|
| TLS ≥1.2 | Терминация на LB, HSTS |
| AD/LDAP, 2FA, SSO | `auth_ldap`, `auth_saml2`, ядровой MFA |
| Шифрование ПД при хранении/передаче | TLS + шифрование диска/БД, политика паролей |
| Вход по разовой JWT-ссылке | Доработка в `local_corplms` |
| Лимит сессий (100), тайм-аут | `limitconcurrentlogins`, session timeout |
| Логирование действий, экспорт | Ядровые logstores + экспорт |
| OWASP (XSS/CSRF/SQLi) | Соблюдается ядром Moodle + ревью кастом-кода |
| Закон РК о ПД, ISO/IEC 27001 | On-prem контур + организационные меры |

## Стек версий
- Moodle 4.5 LTS · PHP 8.3 · PostgreSQL 16 · Redis 7 · Nginx/HAProxy · BigBlueButton 2.7+
