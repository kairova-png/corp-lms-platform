# Как собрать классную дизайн-систему под Moodle-LMS (по порядку)

Контекст: дизайн-систему строим в **Claude Design** (claude.ai/design) под наш headless-фронт ученика
(над Moodle), в бренде **KMG PetroChem**. Те же токены потом красят и Moodle-тему. Принцип: **сначала
основа, потом компоненты, потом экраны** — снизу вверх, чтобы всё было единым.

## Шаг 0. Настройка проекта (форма «Set up your design system»)
- **Link code from GitHub:** `https://github.com/kairova-png/corp-lms-platform` (публичный).
- **Blurb:** Corp LMS — обучение сотрудников KMG PetroChem (видеокурсы, тесты, сертификаты).
- **Notes (бренд):** teal `#1896A7`, accent `#30A6A6`, ink `#18303A`, surface `#EEF3F5`, success `#22C55E`;
  шрифты **Montserrat** (заголовки) + **Inter** (текст); тон — корпоративный нефтехим, чисто, скруглённые
  карточки; UI в `frontend/`; ничего не ведёт в Moodle.

## Шаг 1. Основа (foundations) — делать ПЕРВОЙ
Попроси агента собрать и показать на одной странице:
1. **Цвет** — бренд-шкала teal (50→900), семантические роли: `brand / brand-fg / accent / ink / surface /
   muted / success / warning / danger / border`. Проверь контраст текста ≥ 4.5:1.
2. **Типографика** — шкала: H1 32 / H2 24 / H3 20 / body 16 / small 14 / caption 12; Montserrat для
   заголовков, Inter для текста; line-height 1.5.
3. **Spacing** — шкала 4/8/12/16/24/32/48.
4. **Радиусы** — 8 (кнопки) / 12–16 (карточки) / 999 (пилюли).
5. **Тени** — 1–2 мягкие (sm, md), без «глянца».
6. **Движение** — 150–250ms, ease; уважать `prefers-reduced-motion`.
> Пример промпта: «Собери foundations: teal-палитру от #1896A7, семантические роли, типошкалу на
> Montserrat+Inter, spacing 4–48, радиусы, 2 тени. Покажи как style guide».

## Шаг 2. Базовые компоненты (примитивы) — снизу вверх
Проси по одному, **с вариантами и состояниями** (default/hover/focus/disabled/loading):
Button (primary/secondary/ghost/danger, 3 размера) · Input · Select · Checkbox/Radio · Textarea ·
Badge/Chip · Avatar · Card · Tabs · Tooltip · **Progress (бар + кольцо)** · Modal/Dialog · Toast ·
Table · **Sidebar-nav** · **Topbar**.
> «Сделай Button: варианты primary/secondary/ghost/danger × размеры sm/md/lg, состояния hover/focus/
> disabled/loading, бренд-цвета, радиус 8. Покажи все ячейки на одной карточке».

## Шаг 3. Доменные компоненты LMS (на базе примитивов)
**CourseCard** (обложка-градиент, категория, прогресс, бейдж «Записан») · **CurriculumList / LessonItem**
(иконки типов: видео/текст/тест/задание/сертификат, статус «пройдено») · **VideoPlayer** (плеер +
описание) · **StatTile** (дашборд) · **CertificateCard** (превью + «Скачать PDF») · **QuizQuestion**
(одиночный/множественный/соответствие…) · **AssignmentSubmit** (файл/текст, статус, оценка) ·
**EmptyState/ComingSoon**.

## Шаг 4. Экраны (композиция из компонентов)
В порядке ценности: **Дашборд** → **Каталог** → **Моё обучение** → **Страница курса** (плеер +
программа) → **Тест (прохождение)** → **Профиль** → **Сертификаты** → **Календарь / Оценки** →
**Уведомления**. Каждый экран — desktop + mobile.
> «Собери экран „Страница курса“: слева видео-плеер и описание урока, справа программа (разделы→уроки
> с иконками и прогрессом). Используй наши компоненты VideoPlayer, CurriculumList, ProgressBar».

## Шаг 5. Состояния и адаптив
Для каждого экрана: загрузка (skeleton), пусто, ошибка. Брейкпоинты 375 / 768 / 1024 / 1440. На мобиле
сайдбар → нижнее меню или бургер.

## Шаг 6. Доступность (обязательно — ТЗ п.9/12)
Контраст ≥ 4.5:1 · видимый focus-ring · тач-зоны ≥ 44px · клавиатура · тексты на **RU/KZ** · не цвет
единственный индикатор статуса.

## Шаг 7. Единство и передача в код
- Один источник токенов; компоненты переиспользуются (не рисовать кнопку заново на каждом экране).
- Когда нравится — **пришли мне ссылку проекта** (`claude.ai/design/p/<id>`). Я читаю её через DesignSync
  и реализую 1:1 в коде фронта (и переношу токены в Moodle-тему для единого бренда).

## Подсказки по работе с агентом Claude Design
1. **Будь конкретен**: называй токены/размеры/состояния, а не «сделай красиво».
2. **Снизу вверх**: foundations → примитивы → доменные → экраны. Не начинай с экранов.
3. **Проси варианты и состояния** сразу — иначе агент даст один кейс.
4. **Ссылайся на уже сделанные компоненты** при сборке экранов — так система остаётся единой.
5. **Итерируй точечно**: «увеличь радиус карточки до 16, тень мягче» — маленькими правками.
