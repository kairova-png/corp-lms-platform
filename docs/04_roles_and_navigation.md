# Роли и навигация нового интерфейса (headless)

Требование: **всё внутри нового дизайна, без переходов в Moodle** — для ролей, которые видят новый UI.

## Какие роли видят новый дизайн (предложение)

| Роль | Где работает | Почему |
|---|---|---|
| **Ученик** | 🟢 Полностью новый UI | Это «пользователь». Вся поверхность доступна через mobile-WS (как в офиц. приложении) → без редиректов реально |
| **Куратор / Менеджер обучения** (= Специалист + Контроль обучения из ТЗ) | 🟡 Новый UI для проверки/мониторинга/отчётов; **авторинг контента — в Moodle** | Грейдинг и аналитика отдаются по WS; создание курсов/банка вопросов WS почти не отдаёт |
| **Администратор** | ⚪ Moodle-админка (back-office) | Переписывать админ-панель Moodle = переписать половину Moodle. Не нужно «пользователю» |

> Вывод: «всё в новом дизайне» **полностью достижимо для УЧЕНИКА** и **частично для куратора** (проверка/отчёты — да, авторинг — в Moodle). Админка остаётся служебной.

## Меню и страницы — Ученик (полный паритет, всё в новом UI)
1. **Главная / дашборд** — «Продолжить обучение», ближайшие дедлайны, прогресс, новости.
2. **Каталог курсов** — все доступные, фильтры (категория/компетенция/длительность), запись.
3. **Моё обучение** — мои курсы и программы, статусы.
4. **Страница курса** — программа (разделы→уроки) + плеер.
   - **Видеоурок** (плеер), **текст/лонгрид**, **документ-вьюер** (pdf/pptx/docx), **файлы для скачивания**.
   - **Тест** — прохождение в новом UI: попытки, таймер, навигация, все типы вопросов, результат + review (`mod_quiz_*`).
   - **Задание** — сдача (онлайн-текст/файл), статус, оценка и фидбек (`mod_assign_*`).
   - **Опрос/анкета** — прохождение (`mod_feedback_*` / questionnaire).
   - **SCORM** — плеер (`mod_scorm_*`).
5. **Календарь** — события (курсы, вебинары, дедлайны).
6. **Оценки** — по курсам (`gradereport_*`).
7. **Сертификаты** — список + просмотр/скачивание PDF (`mod_customcert_*`).
8. **Достижения / бейджи / рейтинг**.
9. **Уведомления / Сообщения** (`core_message_*`).
10. **Профиль** — данные, доп. поля, фото, редактирование.
11. **Библиотека / документы** — общее файловое хранилище.

## Меню и страницы — Куратор/Менеджер (фаза 2)
- **Проверка заданий и тестов** (грейдинг, комментарии) — `mod_assign_*`.
- **Мониторинг прогресса** учеников/групп (наставничество).
- **Отчёты и аналитика** (охват, прогресс, KPI, по компетенциям) — отчётные WS + наш `local_corplms`.
- **Назначение обучения** (курсы/программы группам).
- *(Авторинг курсов/уроков/банка вопросов — в Moodle-админке.)*

## Что нужно для «без редиректов»
- Все интерактивы ученика — через **mobile-WS** (включить уже сделано). Часть write-функций требует включения в сервисе.
- Файлы/видео/картинки из Moodle — через `pluginfile` с токеном (проксировать на сервере фронта, чтобы токен не светился).
- Это **большой объём** (особенно плеер тестов и все типы вопросов) → строим фазами, начиная с ученика.

## Решение, которое нужно от заказчика
- **Объём ролей в новом UI:** только Ученик · Ученик + Куратор · Все (вкл. админку — не рекомендуется).
- После выбора — фиксируем меню роли и идём страница за страницей.
